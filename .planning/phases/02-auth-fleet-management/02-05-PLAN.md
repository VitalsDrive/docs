<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Invite token → org_invites table | Token inserted via anon key + RLS; only org owner can insert |
| Invite link URL → Browser clipboard | navigator.clipboard requires HTTPS in production |
| /join?token= route → org membership | Token lookup must validate expiry and single-use status before joining |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-05-01 | Elevation of Privilege | org_invites INSERT | mitigate | RLS INSERT policy on org_invites: only org owner (organizations.owner_id = auth.jwt()->>'sub') can create invites |
| T-02-05-02 | Elevation of Privilege | invite token brute-force | mitigate | Token is crypto.randomUUID() — 128-bit entropy; expires in 7 days; single-use invalidates on first use |
| T-02-05-03 | Spoofing | onboarding owner_id set incorrectly | mitigate | organization.service.ts and fleet.service.ts patched to use Auth0 sub (not Supabase auth.getUser() which returns null for Auth0 users) |
| T-02-05-04 | Information Disclosure | invite link in clipboard | accept | User explicitly clicks Copy; link is intended to be shared; no PII in token itself |
| T-02-05-05 | Tampering | inviteType tampered on client | accept | type field validated by DB CHECK constraint (type IN ('single-use', 'multi-use')); invalid values rejected by Supabase |
| T-02-05-06 | Repudiation | created_by set from client | mitigate | created_by reads from Auth0 JWT sub via authService — not user-editable input field; RLS INSERT policy also confirms org membership |
</threat_model>

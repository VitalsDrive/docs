    ALTER TABLE organizations ALTER COLUMN owner_id TYPE TEXT USING owner_id::TEXT;

    SECTION 5 — Create org_invites table (D-19, RESEARCH Pattern 8):
    CREATE TABLE IF NOT EXISTS org_invites with columns: id UUID PRIMARY KEY DEFAULT gen_random_uuid(), token TEXT NOT NULL UNIQUE, org_id UUID NOT NULL REFERENCES organizations(id) ON DELETE CASCADE, created_by TEXT NOT NULL (Auth0 sub, not UUID), type TEXT NOT NULL CHECK (type IN ('single-use', 'multi-use')), role TEXT NOT NULL DEFAULT 'viewer' CHECK (role IN ('viewer', 'driver', 'fleet_admin')), expires_at TIMESTAMPTZ NOT NULL, used_at TIMESTAMPTZ.
    After CREATE: ALTER TABLE org_invites ENABLE ROW LEVEL SECURITY;
    Add indexes: CREATE INDEX IF NOT EXISTS idx_org_invites_token ON org_invites(token); idx_org_invites_org_id; idx_org_invites_expires ON org_invites(expires_at).

    - SELECT: owner of the org can read their org's invites (check organizations.owner_id = auth.jwt()->>'sub')

    - SELECT for join link: anyone with valid non-expired token can read it (for the invite landing page)

    SECTION 7 — RLS for users table own-row access:
    ALTER TABLE users ENABLE ROW LEVEL SECURITY;
    CREATE POLICY "Users can read their own row" ON users FOR SELECT USING (external_auth_id = (auth.jwt()->>'sub'));
    CREATE POLICY "Service role can manage users" ON users FOR ALL USING (auth.jwt()->>'role' = 'service_role');

    Do NOT use auth.uid() in users table RLS — use auth.jwt()->>'sub' directly since the external_auth_id stores the Auth0 sub string, not a UUID that maps to auth.users.id.

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01-01 | Elevation of Privilege | org_invites INSERT policy | mitigate | Only org owner (auth.jwt()->>'sub' matches organizations.owner_id) can create invites |
| T-02-01-02 | Elevation of Privilege | users RLS — service_role bypass | mitigate | Service role policy scoped to auth.jwt()->>'role' = 'service_role'; anon users cannot write user rows |
| T-02-01-03 | Spoofing | users.external_auth_id matching JWT sub | mitigate | auth.jwt()->>'sub' comes from verified Auth0 JWT — Supabase validates signature via third-party provider config |
| T-02-01-04 | Information Disclosure | org_invites token in DB | mitigate | Token indexed but SELECT policy restricts reads to org owner; token itself is random UUID (gen_random_uuid) |
| T-02-01-05 | Denial of Service | fleet_members.user_id TYPE change | accept | One-time migration on existing data; production data is minimal at Phase 2 start |

To get the fleet_id for the org: query fleets table for the first fleet in org_id.

Redemption flow (D-22):
1. Read token from URL queryParams
2. Query org_invites: SELECT * WHERE token = ? AND expires_at > now() AND (type = 'multi-use' OR used_at IS NULL)
3. If not found or expired → show error "This invite link is invalid or has expired."
4. Get current user's Auth0 sub from authService
5. Query first fleet for the org: SELECT id FROM fleets WHERE organization_id = org_id LIMIT 1
6. Insert into fleet_members: { user_id: authSub, fleet_id, role: invite.role } (D-22: role set by inviter at generation, default 'viewer'; invitees never join as 'owner')
7. If single-use: UPDATE org_invites SET used_at = now() WHERE token = ?
8. Redirect to /onboarding/org (so new member goes through onboarding with their new org context)

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| /join?token= URL → org_invites | Token read from URL queryParam; validated against DB before any membership change |
| fleet_members insert → Supabase RLS | RLS on fleet_members must allow insert for authenticated users with valid token |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-06-01 | Elevation of Privilege | Token enumeration | mitigate | Token is crypto.randomUUID() (128-bit entropy); expires in 7 days; single-use invalidated on first redemption |
| T-02-06-02 | Elevation of Privilege | fleet_members insert role | mitigate | Role is read from org_invites.role (set at invite generation); options are viewer/driver/fleet_admin — never 'owner'. Owner role cannot be granted via invite link. |
| T-02-06-03 | Spoofing | Unauthenticated token redemption | mitigate | Component checks isAuthenticated() before insert; redirects to login if unauthenticated, preserving return URL |
| T-02-06-04 | Repudiation | Token reuse after single-use | mitigate | UPDATE used_at immediately after successful insert; subsequent SELECT filters WHERE used_at IS NULL |
| T-02-06-05 | Denial of Service | Replay of expired tokens | accept | expires_at filter in SELECT query; expired tokens return 'invalid or expired' error |
</threat_model>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser form → Supabase vehicles table | Vehicle form data inserted via anon key + RLS; RLS restricts to org members |
| Vehicle soft-delete → Supabase | Status update rather than DELETE; historical telemetry preserved (UI-SPEC copy) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-04-01 | Elevation of Privilege | Vehicle CRUD via anon key | mitigate | RLS policies from migration 006 restrict vehicle access to fleet_members of the org; anon key alone cannot bypass RLS |
| T-02-04-02 | Tampering | vehicle nickname form input | mitigate | Angular Validators.required + maxLength(40) per UI-SPEC form validation; backend RLS enforces org ownership |
| T-02-04-03 | Tampering | vehicle year input | mitigate | Validators.pattern(/^\d{4}$/) rejects non-numeric and non-4-digit values; form shows inline error per UI-SPEC |
| T-02-04-04 | Information Disclosure | loadAllVehicles returns other orgs' vehicles | mitigate | RLS fleet_members EXISTS policy scopes query to authenticated user's org — empty array if RLS fails, not cross-org data |
| T-02-04-05 | Denial of Service | Soft delete vs hard delete | accept | Soft delete preserves historical telemetry; inactive vehicles excluded from active queries by .eq('status', 'active') filter |
</threat_model>

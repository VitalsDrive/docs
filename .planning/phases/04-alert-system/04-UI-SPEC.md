## Interaction Contracts

### Toast on New Alert Arrival
- Trigger: Supabase Realtime INSERT event on `alerts` table received by `AlertService`.
- Content: 4px severity-colored left bar + severity icon + uppercase type label + vehicle name (truncated) + alert message + "View" link to `/alerts`.
- Duration: critical alerts — persistent until manually dismissed. Warning alerts — auto-dismiss after 6000ms.
- Max visible toasts: 3. If overflow: existing `.toast-overflow` "N more alerts" counter shows.
- Animation: `slide-in-top` (existing keyframe) — slide from `translateY(-8px)` to `0`, 220ms `--vd-ease-out`.
- Dismiss: `×` button top-right of toast. `aria-label="Dismiss alert"`.
- "View" link: navigates to `/alerts`. Does NOT dismiss toast — user must dismiss explicitly.
- `prefers-reduced-motion`: animation `none` — toast appears instantly.

### Bell Badge (HeaderComponent)
- Badge shows when `unacknowledgedCount() > 0`. Hidden when count = 0 (via `matBadgeHidden`).
- Badge content: integer count. If count > 99: show "99+" (Angular Material mat-badge handles overflow via `matBadgeMax="99"`).
- Click: navigate to `/alerts` (router link, no modal).
- Badge color: `warn` (maps to `--vd-critical` red via Material theme).
- No animation on badge itself — count updates reactively via signal.

### Vehicle Card Alert Badge
- Badge renders in card-header-right area via `mat-badge` on an icon element.
- Shows unacknowledged count for that specific vehicle.
- Hidden when count = 0.
- Card border class: `.card--critical` when highest severity = critical; `.card--warning` when highest = warning; no alert class when count = 0.
- Existing critical card has `box-shadow: 0 0 16px rgba(239, 68, 68, 0.2)` — retain unchanged.

### Map Marker Color + Pulse
- Color determination order (highest severity wins):
  1. Has unacknowledged critical alert → red (#ef4444)
  2. Has unacknowledged warning alert → yellow (#eab308)
  3. No unacknowledged alerts, vehicle running → lime (#84cc16)
  4. Offline / stale → muted umber (#5a4530) — Phase 3 rule, unchanged
- Pulse: `.pulse` class applied when any unacknowledged alert exists (critical or warning). Removed when count = 0.
- Existing `pulse-ring` keyframe (1.5s ease-in-out infinite) is correct — reuse as-is.
- `prefers-reduced-motion`: suppress `.pulse` animation class.

### `/alerts` Page — Alert List
- Default view: all alerts last 7 days, newest first. Unacknowledged rows rendered at full opacity; acknowledged rows at opacity: 0.45.
- Filter chips: "All", "Critical", "Warning", "Acknowledged" — single-select. Active chip uses `.filter-chip--active` (amber border + text). Count badge on "Critical" chip when critical unacknowledged count > 0 (red badge).
- Page size: 25 rows per page. `mat-paginator` at bottom.
- Acknowledge button: `mat-stroked-button`, label "Acknowledge", visible only on unacknowledged rows. On click: optimistic opacity transition to 0.45, button disappears, Supabase UPDATE fires in background. On error: revert opacity, show snack-bar error.
- Loading state: while `alertResource` status is `'loading'` — centered `mat-spinner` (existing pattern). No skeleton.
- Error state: show error banner (same `.error-banner` pattern as Phase 3 — red border + background + message + retry).

### Acknowledged Alert Transition
- Duration: 300ms, ease-out.
- Property: `opacity` from `1` to `0.45`.
- No layout shift — row stays in place. No collapse or removal animation.
